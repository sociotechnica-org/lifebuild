# Job 9: Betting Table

Review the content backlog and decide what gets appetite. This is the DECIDE station for content production.

## Trigger

Human asks "What content should we bet on next?" or when the weekly review flags shaped items ready in the backlog.

## Key Principle

From Shape Up: Not everything shaped gets built. The betting table is where you decide what's worth the investment. Each bet gets an appetite (time budget). If it can't be done within appetite, it's the wrong shape.

## Procedure

1. Read the content backlog (location TBD — depends on editorial calendar design)
2. Read the content pillars — does each candidate align with a pillar?
3. Read the acquihire brief — which value channels does each candidate serve?

4. **Review shaped items:**

   For each shaped content idea, assess:
   - **Pillar alignment:** Which content pillar does it serve?
   - **Value channel alignment:** Which of the three acquihire channels does it advance?
   - **Channel-market fit:** Will this reach the right people on the planned channel?
   - **Appetite:** How much time should this take? (LinkedIn post: 1 hour. Blog post: 2 days. Video: 1 week.)
   - **Opportunity cost:** What are we NOT doing if we do this?

5. **Place bets:**
   - Select 1-3 items for the next cycle
   - Assign appetite to each
   - Identify who does what (human vs. AI production)
   - Set circuit breaker: if not done within appetite, kill or reshape

6. **Archive unbetted items:**
   - Items not selected stay in backlog
   - Items that have been passed over 3+ times: consider killing or reshaping

## Output

Decisions recorded in a session note at `.context/arvin-sessions/[date]-betting-table.md`, or as a section within the weekly review.
