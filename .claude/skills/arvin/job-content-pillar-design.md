# Job 3: Content Pillar Design

Define the 3-5 content pillars that anchor all content production. Pillars must exist before channel strategy or editorial calendar can be designed.

## Trigger

Human asks Arvin to define content pillars, or after Master Strategy is complete and pillars need to be specified.

## Procedure

1. Read the master communications strategy
2. Read the acquihire brief — what topics serve the three value channels?
3. Read the research document — content pillar framework section
4. Work with the human to identify 3-5 pillars:

   **For each candidate pillar, assess:**
   - Does it serve at least one acquihire value channel?
   - Can it sustain multiple pieces of content? (Not a one-off topic)
   - Does it play to byline authority? (Is this something Danvers or Jess has genuine conviction about?)
   - Does it differentiate? (Is this a position, not just a topic?)

   **Assessment framework:**
   - Pillar should be opinionated, not educational (37signals model)
   - Pillar should document what you're doing, not philosophize (build in public principle)
   - Each pillar should have natural cluster topics that branch from it

5. For each approved pillar, define:
   - The core position (what we believe, stated as a claim)
   - 5-10 cluster topics that branch from it
   - Which value channels it serves
   - Which division(s) it feeds (1:Many primarily, but some pillars may feed 1:1 or Groups content)

## Output

A source document at `docs/context-library/sources/content-pillars-[date].md`.
