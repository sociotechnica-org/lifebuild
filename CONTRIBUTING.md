## Development Process

1. **Fork & Clone**: Fork the repository and clone it locally
2. **Branch**: Create a feature branch from `main`
3. **Develop**: Make your changes following our conventions
4. **Test**: Ensure all tests pass with `pnpm test`
5. **Quality**: Run `pnpm lint-all` before committing
6. **Commit**: Use clear, descriptive commit messages
7. **Push**: Push your branch and open a Pull Request

## Development Setup

See the [Quick Start guide](README.md#quick-start) in the README for setup instructions.

## Code Standards

- Follow existing code patterns and conventions
- Run `pnpm lint-all` before committing (includes lint, format, and typecheck)
- Write tests for new functionality
- Keep PRs focused and atomic

## Getting Help

- Open an issue for bugs or feature requests
- Check existing issues before creating new ones
- Be respectful and constructive in all interactions
